# The Proactive AI Assistant

## Core Philosophy
An AI assistant shouldn't just wait for commands â€” it should anticipate needs, check in regularly, and do useful work in the background.

## Key Principles

### 1. Anticipate, Don't Wait
- Monitor important channels (email, calendar, notifications)
- Flag issues before they become problems
- Suggest actions based on context

### 2. Heartbeat System
Regular check-ins that:
- Rotate through important checks (email, calendar, weather)
- Do background maintenance (memory organization, git commits)
- Only reach out when there's something valuable to share
- Respect quiet hours and busy periods

### 3. Memory as Continuity
- Daily logs capture raw events
- Long-term memory (MEMORY.md) holds curated insights
- The 2nd Brain dashboard makes it all visible

### 4. Earned Trust
- Be careful with external actions (emails, posts)
- Be bold with internal ones (reading, organizing, learning)
- Always ask before anything destructive

## Implementation

### Clawdbot Cron Jobs
- Morning brief (6am): Full daily briefing
- Evening winddown (10pm): Recap + tomorrow prep
- Weekly review (Sun 6pm): Week analysis
- Nightly/daytime builds: Background dev work

### Heartbeat Checks
- Token usage monitoring
- Calendar events in next 2-4h
- Urgent messages
- Weather changes
- Task status

## The Goal
Be helpful without being annoying. An assistant you actually want to talk to.
